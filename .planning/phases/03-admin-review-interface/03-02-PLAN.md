  <name>Task 1: Create review.html with post list, split-pane editor, figure display, approve/reject actions, and pipeline trigger</name>
  <files>admin/static/review.html</files>
  <action>
Create `admin/static/review.html` as a self-contained HTML page using the same styling conventions as `index.html` (system-ui font, max-width 1000px, card sections with white bg and border, blue primary buttons, status badges). Include `marked.js` via CDN: `<script src="https://cdn.jsdelivr.net/npm/marked/marked.min.js"></script>`.

**Layout structure (3 sections):**

1. **Navigation bar** at top: Simple nav with links to "Discovery" (href="/") and "Review" (href="/review.html", current page highlighted). Style: flex row with gap, current page bold/underlined.

2. **Pipeline trigger section** (card): "Run Full Pipeline" button that POSTs to `/api/pipeline/run`, shows running/complete/error status with polling at 2s interval (same pattern as discovery in index.html). Display step progress (discovery, generation) and final result (papers discovered, posts generated).

3. **Posts list section** (card): Fetches from `GET /api/posts` on page load. Renders a table with columns: Paper Title, Status (badge), Word Count, Actions. Status badges styled same as index.html (draft=yellow, approved=green, rejected=red). Each row has a "Review" link/button that loads the post detail view. Include filter buttons at top: "All", "Drafts", "Approved", "Rejected" -- clicking filters calls `GET /api/posts?status=draft` etc.

4. **Post detail view** (initially hidden, shown when a post is selected): Replaces or overlays the list view. Contains:
   - **Header**: Post title, paper arxiv_id (linked to arxiv), status badge, word count. "Back to list" button.
   - **Figures section**: Displays extracted figures as `<img>` tags using the URL from the API response (`figure.url`). Show "No figures extracted" if empty. Style figures in a horizontal scrollable row, max-height 250px.
   - **Split-pane editor**: Left side is a `<textarea>` with the raw markdown content. Right side is a `<div>` that renders the markdown as HTML via `marked.parse()`. The preview updates on every `input` event on the textarea. Style: flex row, each pane 50% width, textarea with monospace font, preview with rendered HTML styling (headings, paragraphs, lists, code blocks).
   - **Action buttons row**: "Save Changes" (blue, calls PUT /api/posts/{arxiv_id} with textarea content), "Approve" (green, calls POST /api/posts/{arxiv_id}/approve), "Reject" (red, calls POST /api/posts/{arxiv_id}/reject). Show confirmation dialog before approve/reject. After approve/reject, update the status badge and disable the approve/reject buttons. After save, update word count display and show brief "Saved" confirmation.
   - **Concepts section**: Display extracted concepts as small badges/tags below the editor. Each shows concept name and relevance score. Informational only (not editable).

**JavaScript behavior:**
- `loadPosts(status)`: Fetches post list, renders table, handles empty state.
- `viewPost(arxiv_id)`: Fetches post detail from `GET /api/posts/{arxiv_id}`, switches from list to detail view, populates editor and figures.
- `updatePreview()`: Called on textarea input, renders markdown to preview pane via `marked.parse()`.
- `saveContent(arxiv_id)`: PUTs content to API, updates word count, clears unsaved flag.
- `approvePost(arxiv_id)` / `rejectPost(arxiv_id)`: Confirms action, POSTs to API, updates UI.
- `triggerPipeline()` / `pollPipelineStatus(taskId)`: Same pattern as discovery polling in index.html.
- `hasUnsavedChanges` flag: Set true on textarea input, cleared on save. `beforeunload` listener warns if unsaved changes exist.
- `escapeHtml(str)`: Same utility function as in index.html.
- `backToList()`: Returns to list view, warns if unsaved changes.

**CSS within the page** (no external stylesheet): Match index.html conventions. Additional styles for:
- `.editor-container`: flex row, gap 1rem
- `textarea#editor`: 50% width, min-height 400px, font-family monospace, padding 1rem, border 1px solid #e0e0e0, border-radius 6px, resize vertical
- `#preview`: 50% width, min-height 400px, padding 1rem, border 1px solid #e0e0e0, border-radius 6px, overflow-y auto, background #fff
- Preview content styling: h1/h2/h3 margins, p margins, ul/ol margins, code background, blockquote border-left
- `.figures-row`: flex row, gap 1rem, overflow-x auto, padding 0.5rem 0
- `.figure-img`: max-height 250px, border-radius 4px, border 1px solid #e0e0e0
- `.concept-tag`: inline-block, padding 0.2rem 0.5rem, background #f0f0f0, border-radius 4px, font-size 0.8rem, margin 0.2rem
- `.badge-draft`: background #fef3c7, color #92400e
- `.badge-approved`: background #d1fae5, color #065f46
- `.badge-rejected`: background #fee2e2, color #991b1b
- `.btn-approve`: background #059669, color white
- `.btn-reject`: background #dc2626, color white
- `.btn-save`: background #2563eb, color white
- Action buttons: padding 0.5rem 1rem, border none, border-radius 6px, cursor pointer, font-size 0.9rem
