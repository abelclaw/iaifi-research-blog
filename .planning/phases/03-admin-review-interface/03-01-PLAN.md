  <action>
1. In `pipeline/models.py`, add a `BlogPostStatus` enum with values DRAFT="draft", APPROVED="approved", REJECTED="rejected". Place it after PaperStatus.

2. In `pipeline/db.py`, add three new methods to the Database class:
   - `get_blog_posts(self, status: str | None = None) -> list[dict]`: Query blog_posts table, optionally filtered by status, ordered by created_at DESC. Use aiosqlite.Row factory, return list of dicts. Join with papers table to include paper title and iaifi_category in results (SELECT bp.*, p.title as paper_title, p.iaifi_category FROM blog_posts bp LEFT JOIN papers p ON bp.paper_arxiv_id = p.arxiv_id).
   - `update_blog_post_content(self, arxiv_id: str, content: str) -> None`: Update blog post content for the given paper arxiv_id. Recalculate word_count as `len(content.split())` server-side. Set updated_at to datetime('now'). This prevents word count drift (Pitfall 5 from research).
   - `update_blog_post_status(self, arxiv_id: str, status: str) -> None`: Update blog post status field and updated_at timestamp.

3. Create `admin/routes/posts.py` following the exact pattern from `admin/routes/papers.py`:
   - `PostUpdateRequest(BaseModel)` with field `content: str`
   - `GET /api/posts` - list posts, optional `?status=` query param. Uses `db.get_blog_posts()`.
   - `GET /api/posts/{arxiv_id}` - get post detail. Returns `{"post": post_dict, "figures": figures_list}`. For each figure, transform `figure_path` to a URL: strip the FIGURES_DIR prefix (from `settings.FIGURES_DIR`) and prepend `/figures/`. If figure_path does not start with FIGURES_DIR, split on "figures/" and reconstruct. This prevents Pitfall 2 (path mismatch). Also fetch and include concepts via `db.get_concepts(arxiv_id)` in the response as `{"post": ..., "figures": ..., "concepts": ...}`.
   - `PUT /api/posts/{arxiv_id}` - update content. Validate post exists (404 if not). Call `db.update_blog_post_content()`. Return `{"status": "updated", "word_count": len(body.content.split())}`.
   - `POST /api/posts/{arxiv_id}/approve` - approve draft. Validate post exists (404) and status is "draft" (400 if not). Call `db.update_blog_post_status(arxiv_id, "approved")`. Return `{"status": "approved"}`.
   - `POST /api/posts/{arxiv_id}/reject` - reject draft. Same validation as approve. Call `db.update_blog_post_status(arxiv_id, "rejected")`. Return `{"status": "rejected"}`.

All routes use `Database(db_path=settings.DB_PATH)` instantiated per-request, matching existing pattern.
  </action>

   - Module-level `_pipeline_tasks: dict = {}` for in-memory task tracking.
   - `POST /api/pipeline/run` endpoint: Generate task_id as `f"pipeline-{int(time.time())}"`, store initial status in _pipeline_tasks dict, add background task via `BackgroundTasks`, return `{"task_id": task_id, "status": "started"}`.
   - `GET /api/pipeline/{task_id}/status` endpoint: Return current task status from _pipeline_tasks dict. 404 if task_id not found.
   - `run_full_pipeline(task_id: str)` async function (the background task):
     - Update step to "discovery". Create Database + DiscoveryOrchestrator, call `await orchestrator.discover()`.
     - Update step to "generation". Get papers with status "pdf_downloaded" OR papers with status containing "extracted" that do NOT have a blog post yet. For each eligible paper: check `await db.get_blog_post(paper["arxiv_id"])` -- if None, create ContentGenerator and call `await generator.generate_content(paper["arxiv_id"])`, increment counter.
     - On success: update task status to "complete" with result dict containing papers_discovered and posts_generated counts.
     - On exception: update task status to "error" with error string.
   - Import DiscoveryOrchestrator and ContentGenerator from pipeline.orchestrator, Database from pipeline.db, settings from pipeline.config.
