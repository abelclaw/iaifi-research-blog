2. Create `pipeline/generator/llm_client.py` -- LLMClient class:
   - Constructor takes optional model, temperature, cache_dir params. Falls back to LLMSettings defaults from config.
   - At init: set `litellm.cache = Cache(type="disk", disk_cache_dir=cache_dir)`. Track `self.total_cost = 0.0`.
   - `async def generate(self, system_prompt: str, user_prompt: str, max_tokens: int = 4096) -> str`: Call `litellm.acompletion()` with model, messages (system + user), temperature, max_tokens. Extract cost from `response._hidden_params.get("response_cost", 0)` and add to total_cost. Return `response.choices[0].message.content`.
   - `async def generate_structured(self, system_prompt: str, user_prompt: str, response_model: type[BaseModel], max_tokens: int = 4096) -> BaseModel`: Same as generate but pass `response_format=response_model`. Parse response content with `response_model.model_validate_json()`. Wrap in try/except -- on ValidationError, strip markdown code fences (```json ... ```) from content and retry parse. If still fails, raise with clear error.
   - Add retry logic: wrap acompletion calls in a simple retry (max 2 retries) for transient API errors (TimeoutError, connection errors). Log each retry.
   - Check at init that at least one API key env var is set (ANTHROPIC_API_KEY or OPENAI_API_KEY or OPENAI_API_BASE for Ollama). If none set, raise ValueError with clear message about which env vars to set.

3. Create `pipeline/extractor/__init__.py` (empty init file).

4. Create `pipeline/extractor/pdf_reader.py`:
   - Function `extract_paper_text(pdf_path: str, max_chars: int = 10000) -> str`: Use `pymupdf4llm.to_markdown(pdf_path, write_images=False, show_progress=False)` to convert PDF to markdown. Truncate to max_chars if longer, appending "\n\n[Content truncated for processing]". Return the markdown string. Handle FileNotFoundError with a clear error message.

5. Create `pipeline/extractor/figure_extractor.py` -- FigureExtractor class:
   - Constructor takes output_dir (default from settings.FIGURES_DIR), min_size (100px), min_area_ratio (0.02).
   - `def extract_figures(self, pdf_path: str, arxiv_id: str, max_figures: int = 3) -> list[dict]`: Open PDF with pymupdf. For each page:
     - Strategy 1 (raster): `page.get_images(full=True)` -> for each image, `doc.extract_image(xref)` -> check significance (min dimensions, min area ratio vs page area). Store candidate with type="raster", page, width, height, xref, data, ext.
     - Strategy 2 (vector): `page.get_drawings()` -> if paths exist, `page.cluster_drawings(drawings=paths)` -> for each cluster bbox, check area ratio. Render to pixmap at 150 DPI with `page.get_pixmap(clip=box, dpi=150)`. Store candidate with type="vector", page, width, height, pixmap.
   - `_score_candidate(candidate) -> float`: Score by area * aspect_ratio_score * page_position_score. Aspect score: penalize if aspect ratio < 0.3 (narrow strips). Page score: 1.0 / (1 + page * 0.1) -- prefer early pages.
   - `_is_significant(width, height, page_area) -> bool`: width >= min_size AND height >= min_size AND (width*height)/page_area >= min_area_ratio.
   - Sort candidates by score descending, take top max_figures.
   - Save selected figures: create dir `output_dir/arxiv_id/`. For raster: convert to PNG using Pillow (handles JBIG2, JPEG, etc. -> consistent PNG). For vector: save pixmap as PNG. Return list of dicts with path, page, width, height, type keys.
   - Close the PDF document in a finally block.
