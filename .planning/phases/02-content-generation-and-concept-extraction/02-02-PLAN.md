Create the `pipeline/generator/prompts/` directory.

1. Create `blog_draft.j2` (Journalist pass -- initial draft from paper):
   - Accept template variables: title, authors (list), abstract, paper_content (markdown), iaifi_category
   - Display paper information section: title, authors (joined by comma), category
   - Display abstract section
   - Display paper content excerpt section
   - Instructions section: Write an 800-1500 word blog post with three sections:
     - Overview (2-3 paragraphs): Broad, accessible explanation using analogies and everyday language. A scientifically curious non-expert should understand fully.
     - Key Highlights (2-3 paragraphs): Main findings, methods, innovations. Introduce technical terms but always explain on first use. Be specific about novelty.
     - Impact and Implications (1-2 paragraphs): Meaning for the field, future research in AI and/or fundamental physics, broader scientific goals.
   - Important rules: Start accessible then gradually increase technical depth. Never fabricate results. Use active voice. Include specific numbers/methods/results from paper. Do NOT include a title line -- just start with the overview.

2. Create `blog_simplify.j2` (Reader pass -- accessibility refinement):
   - Accept template variable: draft (the journalist's output)
   - Instructions: You are reviewing this science blog post as an educated general reader (not a specialist). Your job is to:
     - Ensure the first two paragraphs (Overview) are genuinely accessible to someone with no background in this field
     - Replace any jargon in the Overview with plain-language alternatives
     - Add brief explanations for any technical terms that appear without context
     - Keep the Key Highlights and Impact sections at their current technical level -- those are for a more technical audience
     - Preserve all specific numbers, methods, and results
     - Return the complete revised post (all three sections)

3. Create `blog_polish.j2` (Editor pass -- final polish):
   - Accept template variables: draft (the reader-revised output), target_min (int, default 800), target_max (int, default 1500)
   - Instructions: You are a senior science editor doing a final polish. Your job is to:
     - Ensure the post follows the structure: Overview, Key Highlights, Impact and Implications (use ## headers for each)
     - Verify the accessibility gradient: overview is broad, highlights are specific, impact connects to field
     - Tighten prose: remove redundancy, improve flow between sections
     - Target word count: {{ target_min }}-{{ target_max }} words. Current draft may be longer or shorter -- adjust by expanding thin sections or trimming verbose ones
     - Ensure section headers are exactly: ## Overview, ## Key Highlights, ## Impact and Implications
     - Return ONLY the final blog post text with section headers. No preamble, no commentary.
