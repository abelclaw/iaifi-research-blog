1. Create `pipeline/generator/prompts/concepts.j2`:
   - Accept template variables: title, authors (list), abstract, paper_content (markdown), iaifi_category
   - Instructions: Analyze this research paper and extract scientific concepts and categorization data.
   - Paper Information section: display title, authors (joined), current category
   - Abstract section: display abstract
   - Paper Content section: display paper_content
   - Extraction instructions:
     - Extract 5-15 key scientific concepts from this paper. For each concept provide: a short name (1-4 words), a one-sentence description of what it means in the context of this paper, and a relevance score (0-1) indicating how central it is to the paper's contribution.
     - Focus on: specific methods/algorithms, theoretical frameworks, physical phenomena, mathematical tools, datasets, and key results.
     - Avoid generic terms like "machine learning" or "physics" unless the paper makes a specific novel contribution to that general area.
     - Determine the primary IAIFI theme: one of "Foundational AI", "Theoretical Physics", "Experimental Physics", or "Astrophysics". The current assigned category is {{ iaifi_category }} -- validate or correct based on paper content.
     - List any secondary themes that are also relevant (can be empty).

2. Create `pipeline/generator/concept_extractor.py` -- ConceptExtractor class:
   - Constructor takes `llm_client: LLMClient` and optional `prompts_dir: str` (default "pipeline/generator/prompts").
   - Initialize Jinja2 Environment with FileSystemLoader.
   - `async def extract_concepts(self, paper: Paper, pdf_text: str) -> ConceptExtractionResult`:
     - Render `concepts.j2` with paper.title, paper.authors, paper.abstract, paper_content=pdf_text, paper.iaifi_category.
     - System prompt: "You are a scientific concept analyst specializing in AI and fundamental physics research. You identify the key technical concepts, methods, and phenomena in research papers. You provide structured, precise analysis."
     - Call `self.llm.generate_structured()` with the system prompt, rendered template, and `response_model=ConceptExtractionResult`.
     - Log extracted concept count and primary theme.
     - Return the ConceptExtractionResult.
   - Import ConceptExtractionResult from pipeline.models. Import Paper from pipeline.models. Import LLMClient from pipeline.generator.llm_client.

1. Extend `pipeline/orchestrator.py` -- Add a `ContentGenerator` class (keep existing DiscoveryOrchestrator untouched):
   - Constructor takes `db: Database`. Internally creates LLMClient (from config), PostGenerator, ConceptExtractor, and FigureExtractor instances.
   - `async def generate_content(self, arxiv_id: str, on_progress: Callable | None = None) -> dict`:
     - Fetch paper from DB: call `self.db.get_papers()` and find the one matching arxiv_id. If not found, raise ValueError.
     - Construct a Paper model from the DB dict (parse authors/categories from JSON if needed).
     - Verify PDF exists at paper's pdf_path. If not, raise FileNotFoundError with message.
     - Progress callback helper: `_progress(step, details)`.
     - Step 1 -- Extract figures: `_progress("extracting_figures", ...)`. Call `self.figure_extractor.extract_figures(pdf_path, arxiv_id)`. For each figure returned, call `self.db.insert_figure(...)`. Update paper status to FIGURES_EXTRACTED.
     - Step 2 -- Extract text: `_progress("extracting_text", ...)`. Call `extract_paper_text(pdf_path)` to get markdown.
     - Step 3 -- Generate blog post: `_progress("generating_post", ...)`. Call `self.post_generator.generate_post(paper, pdf_path)`. Call `self.db.insert_blog_post(...)` with the BlogPost data. Update paper status to POST_GENERATED.
     - Step 4 -- Extract concepts: `_progress("extracting_concepts", ...)`. Call `self.concept_extractor.extract_concepts(paper, pdf_text)`. Call `self.db.insert_concepts(arxiv_id, [c.model_dump() for c in result.concepts])`. Update paper status to CONCEPTS_EXTRACTED.
     - `_progress("complete", ...)` with summary.
     - Return dict with blog_post, figures, concepts, generation_cost.
   - Wrap entire process in try/except, log errors, report via progress callback.

2. Create `admin/routes/generation.py`:
   - Import necessary modules: FastAPI APIRouter, BackgroundTasks, Database, ContentGenerator.
   - Create router = APIRouter(prefix="/api").
   - In-memory dict `_gen_tasks: dict[str, dict]` for tracking generation task status (same pattern as discovery.py).
   - `POST /api/generate/{arxiv_id}`: Accept arxiv_id path param. Create a background task that instantiates Database + ContentGenerator and calls `generate_content(arxiv_id, on_progress=callback)`. The callback updates `_gen_tasks[task_id]`. Return `{"task_id": task_id, "status": "started"}`. Generate task_id as f"gen-{arxiv_id}-{int(time.time())}".
   - `GET /api/generate/{task_id}/status`: Return current status from `_gen_tasks`. Return 404 if task_id not found.
