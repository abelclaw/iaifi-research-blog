  <name>Task 1: Search page with Pagefind UI and build pipeline verification</name>
  <files>site/src/pages/search.astro, site/package.json</files>
  <action>
1. **Create site/src/pages/search.astro:**
   - Import BaseLayout
   - Page title: "Search Papers"
   - Heading: "Search Papers" as h1
   - Container div with `id="search-container"`
   - Load Pagefind UI assets via `<link>` and `<script>` tags pointing to `/pagefind/pagefind-ui.css` and `/pagefind/pagefind-ui.js`
   - Use `is:inline` on the script tags (Pagefind assets are generated post-build, not available at Astro build time)
   - Initialize PagefindUI in a DOMContentLoaded handler:
     ```javascript
     window.addEventListener("DOMContentLoaded", () => {
       new PagefindUI({
         element: "#search-container",
         showSubResults: true,
         showImages: false,
       });
     });
     ```
   - Add a note below the search: "Search across all published papers by title, authors, or concepts."
   - Style the search container with some spacing (Pagefind provides its own CSS for the search UI)

2. **Update site/package.json build script** to include the full pipeline chain:
   - Ensure `"build"` script is: `"npm-run-all -s build:astro build:search"`
   - Add an `"export"` script: `"cd .. && python -m pipeline.export"` (for manual use; the full build can be triggered from the project root)
   - Verify `"build:search"` is: `"npx pagefind --site dist"`

3. **Run the full build to verify Pagefind indexing:**
   ```bash
   cd site && npm run build 2>&1
   ```
   Pagefind should output the number of pages indexed. If no content exists yet (no approved posts), it will index 0 pages -- that is fine. The important thing is that the build chain completes without errors.

4. **Verify Pagefind generated its index:**
   Check `site/dist/pagefind/` directory exists after build. It should contain `pagefind.js`, `pagefind-ui.js`, `pagefind-ui.css`, and index fragment files.

5. **Add the Header "Search" nav link** -- verify it already points to `/search` from Plan 02. If not, update Header.astro.

Note: Pagefind UI CSS may need minor customization to match the site's Tailwind theme. If the default Pagefind styles clash with Tailwind's reset, add a few overrides in global.css:
```css
/* Pagefind UI overrides */
.pagefind-ui__search-input {
  @apply rounded-lg border-gray-300;
}
```
Only add overrides if there is a visible styling issue -- Pagefind's default UI is clean and functional.
