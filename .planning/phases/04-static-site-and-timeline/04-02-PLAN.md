- Props: `post` object with `id` and `data` (title, authors, abstract, theme, published, concepts, arxivUrl)
- Theme color coding using a map:
  - "Foundational AI" -> blue (bg-blue-100 text-blue-800 border-l-blue-500)
  - "Theoretical Physics" -> violet (bg-violet-100 text-violet-800 border-l-violet-500)
  - "Experimental Physics" -> amber (bg-amber-100 text-amber-800 border-l-amber-500)
  - "Astrophysics" -> emerald (bg-emerald-100 text-emerald-800 border-l-emerald-500)
- Card structure: theme badge + date on first row, title as link to `/posts/{post.id}`, truncated author list (show first 3 + "et al." if more), abstract excerpt (line-clamp-3), up to 5 concept tags as small pills
- Left border colored by theme (border-l-4)
- `data-theme` attribute on the `<article>` element for filtering
- Hover: subtle shadow transition

**site/src/components/PostMetadata.astro:**
- Props: `authors: string[]`, `arxivUrl: string`, `pdfUrl: string`, `published: Date`, `abstract: string`, `theme: string`, `concepts: string[]`, `figures: string[]`
- Display: theme badge (color-coded same as PostCard), publication date formatted as "Month Day, Year", full author list (comma-separated), abstract in a slightly muted/indented block, arXiv link (external, opens new tab) and PDF link, concept tags as pills
- Responsive: stack vertically on mobile, comfortable reading width

**site/src/components/ThemeFilter.astro:**
- Five filter buttons: "All", "Foundational AI", "Theoretical Physics", "Experimental Physics", "Astrophysics"
- "All" is active by default (use `data-active` attribute)
- Active state: dark background, white text (`data-[active]:bg-gray-900 data-[active]:text-white`)
- Inline `<script>` for client-side filtering:
  - On button click: update active state, iterate `#timeline article` elements, show/hide based on `data-theme` matching the filter value
  - "all" shows everything
- Buttons styled as rounded pills with border, transition on hover

**site/src/layouts/PostLayout.astro:**
- Props: `title`, `authors`, `arxivUrl`, `pdfUrl`, `theme`, `published`, `abstract`, `figures`, `concepts`

- Prose styling: use Tailwind `prose prose-lg` classes on the content area for readable typography (install `@tailwindcss/typography` -- add to npm install)

Create the two page files:

**site/src/pages/index.astro (Timeline Feed):**
- Import `getCollection` from `astro:content`
- Import BaseLayout, PostCard, ThemeFilter components
- Fetch all posts: `const posts = await getCollection('posts')`
- Sort by published date descending: `posts.sort((a, b) => b.data.published.getTime() - a.data.published.getTime())`
- Render: BaseLayout wrapping ThemeFilter + a `<div id="timeline" class="space-y-6">` containing PostCard for each post
- Add a hero/intro section above the filter: "IAIFI Research Blog" as h1, brief tagline "Exploring the intersection of AI and fundamental physics" as subtitle
- If no posts exist, show a helpful message: "No published posts yet. Approve posts in the admin interface to see them here."

**site/src/pages/posts/[...slug].astro (Dynamic Blog Post Pages):**
- Import `getCollection`, `render` from `astro:content`
- Import PostLayout
- `getStaticPaths()`: fetch all posts, return `posts.map(post => ({ params: { slug: post.id }, props: { post } }))`
- In the page body: destructure `post` from `Astro.props`, call `const { Content } = await render(post)`
- Render PostLayout with all metadata props passed from `post.data`, with `<Content />` in the slot

Important: Use `[...slug].astro` (rest parameter) not `[slug].astro` -- this handles slugs that might contain path separators. Use `post.id` for the slug value (Astro content collections use `id` as the file-based identifier).

After creating pages, verify the build works:
