  <action>
    1. Create `admin/__init__.py` (empty).

    2. Create `admin/routes/__init__.py` (empty).

    3. Create `admin/routes/discovery.py`:
       - APIRouter with prefix="" (routes include /api/ in path)
       - Module-level `_tasks: dict[str, dict]` for in-memory task tracking (sufficient for single-user local app)
       - `POST /api/discover`:
         - Generate UUID task_id
         - Store task entry with status="running", result=None, started_at=datetime
         - Add background task (FastAPI BackgroundTasks) to run discovery
         - Return {"task_id": task_id, "status": "started"}
       - `GET /api/discover/{task_id}/status`:
         - Look up task in _tasks dict
         - 404 if not found
         - Return full task dict (status, result if complete, error if failed)
       - `run_discovery(task_id: str)` background function:
         - Instantiate Database from config, call initialize()
         - Instantiate ArxivClient from config
         - Import scrape_all_papers from iaifi_scraper
         - Create DiscoveryOrchestrator with all three
         - Call await orchestrator.discover()
         - On success: update _tasks[task_id] with status="complete" and result dict (total_scraped, new_papers, metadata_fetched, pdfs_downloaded, errors)
         - On exception: update _tasks[task_id] with status="error" and error message string
         - Use try/except to ensure task status is always updated

    4. Create `admin/routes/papers.py`:
       - APIRouter
       - `GET /api/papers`:
         - Optional query param: status (str, default None) to filter by PaperStatus
         - Optional query param: category (str, default None) to filter by iaifi_category
         - Instantiate Database from config, fetch papers
         - Return list of paper dicts with fields: arxiv_id, title, authors, abstract, iaifi_category, status, published, pdf_path, code_url
         - Parse JSON strings (authors, arxiv_categories) back to lists before returning
       - `GET /api/papers/{arxiv_id}`:
         - Return single paper detail or 404

    5. Create `admin/app.py`:
       - Create FastAPI app with title="IAIFI Blog Admin"
       - Include routers from discovery and papers modules
       - Mount static files: `app.mount("/", StaticFiles(directory="admin/static", html=True), name="static")` -- mount AFTER API routes so /api/* routes take precedence
       - Add startup event to initialize database (ensure tables exist on app start)
       - Add CORS middleware allowing localhost origins (for dev convenience)
       - At bottom: `if __name__ == "__main__": uvicorn.run("admin.app:app", host="0.0.0.0", port=8000, reload=True)`

    IMPORTANT: The static files mount must come AFTER the API router includes, otherwise it will intercept /api/* requests. FastAPI matches routes in registration order.
  </action>
