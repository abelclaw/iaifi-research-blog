  <action>
    1. Create `pipeline/fetcher/__init__.py` (empty).

    2. Create `pipeline/fetcher/iaifi_scraper.py`:
       - Import ScrapedPaper from pipeline.models, CATEGORY_PAGES from pipeline.config
       - ARXIV_ID_RE pattern: `re.compile(r"(\d{4}\.\d{4,5}(?:v\d+)?)")` for extracting IDs from arXiv links
       - `scrape_category_page(url: str, category: str) -> list[ScrapedPaper]`:
         - GET the page with requests (timeout=30, raise_for_status)
         - Parse with BeautifulSoup(html, "lxml")
         - Find all `<a>` tags whose href matches arxiv.org/abs/ pattern
         - For each arXiv link, navigate the surrounding DOM to extract:
           - Title: the nearest bold/italic text (strong > em) above the link
           - Authors: text between the title element and the arXiv link
           - Abstract: text after the link block, often prefixed with "Abstract" in italics
           - Code URL: sibling link to github.com if present
         - Normalize arxiv ID by stripping version suffix: `re.sub(r'v\d+$', '', arxiv_id)`
         - Return list of ScrapedPaper objects
       - `scrape_all_papers() -> list[ScrapedPaper]`:
         - Iterate all 4 CATEGORY_PAGES
         - Deduplicate by arxiv_id (first category wins)
         - Validate: if any category returns 0 papers, log a warning (likely scraper breakage)
         - Return deduplicated list
       - Add logging throughout (use logging module, not print)
       - IMPORTANT: The exact HTML structure may vary. Build the parser to be resilient -- if a field can't be extracted, log a warning and continue with what's available. The arxiv_id is the only required field from scraping (metadata enrichment via arXiv API fills in the rest).

    3. Create `pipeline/fetcher/arxiv_client.py`:
       - Import arxiv library, Path, logging, re
       - `ArxivClient` class:
         - __init__(pdf_dir, delay_seconds) from config defaults
         - Creates arxiv.Client with page_size=100, delay_seconds, num_retries=3
         - Creates pdf_dir with parents=True, exist_ok=True
       - `fetch_metadata_batch(arxiv_ids: list[str]) -> dict[str, arxiv.Result]`:
         - Create arxiv.Search(id_list=arxiv_ids)
         - Iterate self.client.results(search)
         - Key results by base ID (strip version suffix)
         - Log warning for any requested IDs missing from results
         - Return dict[base_id, Result]
       - `download_pdf(result: arxiv.Result) -> Path | None`:
         - Compute filename from short ID (strip version, replace / with _)
         - If file exists and size > 0, return cached path
         - Try download via result.download_pdf()
         - Verify file exists and size > 0 after download
         - On failure: clean up partial file, log error, return None
         - PDF download failure is non-fatal (paper metadata still valuable)

    4. Create `pipeline/orchestrator.py`:
       - `DiscoveryOrchestrator` class, initialized with Database, scraper module, ArxivClient
       - `async def discover(on_progress: Callable | None = None) -> DiscoveryResult`:
         - Step 1: Scrape IAIFI (call scrape_all_papers)
         - Step 2: Get existing IDs from DB (await db.get_all_arxiv_ids())
         - Step 3: Compute new papers = scraped IDs not in existing IDs
         - Step 4: Insert new papers with status=DISCOVERED
         - Step 5: Fetch arXiv metadata in batches of ARXIV_BATCH_SIZE
           - For each batch: fetch_metadata_batch, then for each result:
             - Update paper with full metadata (title from arXiv, authors, abstract, categories, published date, pdf_url)
             - Set status=METADATA_FETCHED
         - Step 6: Download PDFs for all papers with status=METADATA_FETCHED
           - For each: download_pdf, if successful update pdf_path and status=PDF_DOWNLOADED
           - Mark as INGESTION_COMPLETE after PDF downloaded (or PDF_DOWNLOADED if download fails but metadata exists)
         - Step 7: Create and update discovery_run record with totals
         - Call on_progress callback at each major step if provided
         - Return DiscoveryResult with counts and any error messages
       - Handle errors gracefully: individual paper failures should not abort the entire run. Collect errors in a list and continue.
  </action>
